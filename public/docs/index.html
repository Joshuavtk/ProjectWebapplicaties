Introduction

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

localhost:8080

Authenticatie

Basic login

This is used for first time authentication.

Example request:
curl --request POST \
    "localhost:8080/authentication" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"development@dxmusic.nl\",
    \"password\": \"velit\"
}"

const url = new URL(
    "localhost:8080/authentication"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "development@dxmusic.nl",
    "password": "velit"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200):

Gebruiker registratie

een gebruiker kan een account aanmaken bij ons. als dit request successvol is krijgt de gebruiker een validatie code in de mail. De validatie code moet naar de validate endpoint gepost worden voor dat het account actief wordt. de gebruiker krijgt hier de meest basis rol.

Example request:
curl --request POST \
    "localhost:8080/authentication/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"user@project.local\",
    \"password\": \"asdasd\"
}"

const url = new URL(
    "localhost:8080/authentication/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "user@project.local",
    "password": "asdasd"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200):


"success": {
 "message": "Successful"
}
 @response 422 "error": {
 "code": 422,
 "message": "Invalid request."
